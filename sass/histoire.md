# Histoire de Sass

## Introduction
Sass (Syntactically Awesome Stylesheets) est un préprocesseur CSS créé pour rendre l’écriture de styles plus simple, plus organisée et plus puissante.

## Création
Sass a été créé en 2006 par Hampton Catlin et développé ensuite par Natalie Weizenbaum.
À l’origine, il faisait partie du framework Ruby on Rails.

## Évolution
- 2006 : Première version avec une syntaxe indentée (sans accolades ni points-virgules)
- 2010 : Introduction de SCSS, une syntaxe compatible avec le CSS classique
- 2014+ : Adoption massive par les développeurs front-end
- 2020 : Fin officielle de LibSass (ancienne implémentation)

## Sass aujourd’hui
Aujourd’hui, Sass est principalement utilisé via Dart Sass, la version officielle maintenue.
Il est compatible avec tous les environnements modernes (Node.js, bundlers, etc.).

## Pourquoi Sass est populaire
- Variables
- Imbrication (nesting)
- Mixins et fonctions
- Code plus lisible et maintenable

## Conclusion
Sass est devenu un standard dans le développement front-end, facilitant la gestion de projets CSS complexes.
